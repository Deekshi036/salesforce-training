# Salesforce Platform Basics

---

# 1. What is Salesforce Platform?

Salesforce Platform is a cloud-based platform used to build and manage business applications.

It allows companies to:
- Manage customer data
- Build applications
- Automate workflows
- Customize business processes

The platform provides tools for both administrators and developers.

---

# 2. Salesforce Architecture Overview

Salesforce works on a multi-tenant cloud architecture.

This means:
- Multiple companies use the same platform
- Data remains secure for each company
- Resources are shared efficiently

Salesforce provides:
- Security
- Scalability
- Customization
- Cloud accessibility

---

# 3. Important Salesforce Concepts

## App

An App is a collection of related tools, objects, and tabs designed for specific business purposes.

Example:
Sales App

The Sales App contains:
- Leads
- Accounts
- Contacts
- Opportunities

Apps help users organize their work efficiently.

---

## Object

Objects are database tables used to store information.

Salesforce provides standard objects such as:
- Account
- Contact
- Lead
- Opportunity

Objects contain records and fields.

Example:
Account object stores company information.

---

## Tab

Tabs are used to access objects and features inside Salesforce.

Example:
Clicking the Accounts tab opens the Account object.

Tabs help users navigate easily through Salesforce.

---

# 4. Difference Between App and Object

| App | Object |
|---|---|
| Collection of tools and objects | Stores data |
| Used for navigation and organization | Used for managing records |
| Example: Sales App | Example: Account Object |

---

# 5. Configuration vs Coding

## Configuration (No Code)

Configuration means customizing Salesforce using built-in tools without programming.

Examples:
- Creating custom fields
- Modifying page layouts
- Creating validation rules
- Changing tabs and apps

Advantages:
- Easy to implement
- Faster development
- No programming knowledge required

---

## Coding (Apex)

Coding is used when advanced business logic is required.

Salesforce uses Apex programming language.

Examples:
- Complex automation
- Custom calculations
- API integrations
- Advanced workflows

Advantages:
- More flexibility
- Advanced customization
- Handles complex requirements

# 9. Questions and Answers

## What is an App in Salesforce?
An App is a collection of tools, tabs, and objects used for specific business activities.

---

## What is an Object?
An Object is a database table used to store information.

---

## Difference Between App and Object
Apps organize features, while objects store records and data.

---

## What is Multi-Tenant Architecture?
It is a cloud architecture where multiple companies share the same platform securely.

---

## When Should We Use Configuration Instead of Code?
Configuration should be used when simple customization can solve the requirement without programming.

---

## How Does Salesforce Allow Developers to Extend Functionality?
Salesforce allows developers to use Apex, APIs, Lightning Components, and automation tools for advanced customization.
